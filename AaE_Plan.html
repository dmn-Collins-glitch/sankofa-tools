Build-first rule

The Iteration Protocol

A plan that doesn't update is a plan that goes stale. The AI/HITL ecosystem is moving fast — new models, new frameworks, retiring certs, shifting job titles. This protocol bakes in deliberate checkpoints so the plan stays current. Scan, log, adjust.

MON

Weekly Ecosystem Scan

~20 min every Monday
Green signal (note it, no change)

New model release, minor API update, interesting new project, new DL.AI course

Yellow signal (discuss, maybe adjust)

Framework dominance shift, cert expiry announced, job posting pattern change

Red signal (act now)

Cert retiring (like ML Spec), major new tool replacing what you're building toward, hiring freeze in target category

Skip entirely

Twitter hype without GitHub, anything about "superintelligence timeline," vague AI news with no technical substance

1st

Monthly Plan Review

~45 min, first Monday of month

Immediate plan adjustment triggers

  • Any cert on the active stack announces retirement or major restructure
  • A new Anthropic developer certification launches
  • AWS Bedrock adds a feature that makes Project 5 (CloudMind) significantly easier or obsolete
  • A new dominant eval framework emerges that makes EvalKit less relevant
  • Job posting data shows a strong shift toward a skill not in the current plan (e.g., LangGraph becomes table stakes)
  • You're consistently hitting weekly targets 2+ weeks ahead — accelerate the timeline
Dashboard integration

The daily dashboard (streak tracker, session log, heatmap) runs alongside this plan — not separately. Monday scan results get logged as a session note. Monthly review triggers a dashboard reflection entry. Phase transitions are milestone markers on the heatmap. The two tools talk to each other: the dashboard shows what you did, this plan shows where it's going.
